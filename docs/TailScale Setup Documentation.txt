=====================================================
ENROLLPRO TEAM MESH: TAILSCALE BACKEND + DATABASE SETUP
=====================================================
Scope: Development only (trusted teammates on Tailscale)
Default API Port: 5000
Default DB Port: 5432
=====================================================

MODERN WAY (RECOMMENDED)
- Use Tailscale as a secure Mesh VPN.
- Use each teammate's stable Tailscale hostname (for example: buru-degree.ts.net) instead of raw 100.x.x.x IPs.
- Hostnames remain stable even when teammates change Wi-Fi networks.

0) REPO-SPECIFIC CONTEXT
- Frontend env key is VITE_API_URL (Vite), not REACT_APP_API_URL.
- Backend DB env key is DATABASE_URL (Prisma/PostgreSQL).
- API routes are mounted under /api (health endpoint: /api/health).
- Backend currently listens on 0.0.0.0 in server/src/server.ts.

1) ROLE A: SERVER HOST (YOU RUN API + POSTGRES)

Step 1: Join Tailscale
1. Download Tailscale: https://tailscale.com/download
2. Sign in and confirm status is Connected.
3. Find your Tailscale hostname (for example: your-device-name.ts.net).
4. Optionally note your Tailscale IPv4 (usually 100.x.x.x) as fallback.
5. Share hostname (and fallback IP) with teammates.

Step 2: Open Windows Firewall (Run PowerShell as Administrator)

# Allow API traffic on port 5000
New-NetFirewallRule -DisplayName "Tailscale_Node_5000" -Direction Inbound -LocalPort 5000 -Protocol TCP -Action Allow -Profile Any

# Allow PostgreSQL traffic on port 5432
New-NetFirewallRule -DisplayName "Tailscale_Postgres_5432" -Direction Inbound -LocalPort 5432 -Protocol TCP -Action Allow -Profile Any

Step 3: Confirm backend bind host and port
In server/src/server.ts, keep this behavior:

const PORT = process.env.PORT || 5000;
app.listen(PORT as number, "0.0.0.0", () => {
    console.log(`[Server] Running on http://localhost:${PORT}`);
});

Step 4: Configure backend env (server/.env)
Minimum values:

PORT=5000
CLIENT_URL="http://localhost:5173"
DATABASE_URL="postgresql://postgres:postgres@localhost:5432/enrollpro"

Step 5: Allow PostgreSQL to accept Tailscale clients
1. Open your PostgreSQL data directory (for example C:\Program Files\PostgreSQL\<version>\data).
2. In postgresql.conf set:
     listen_addresses = '*'
3. In pg_hba.conf add this line at the bottom:
     host    all             all             100.64.0.0/10            md5
4. Restart PostgreSQL service from Windows Services.

Step 6: Start your services
From repository root:

pnpm --filter server dev

or run both client and server:

pnpm run dev


2) ROLE B: CLIENT TEAMMATE (YOU CONNECT TO SOMEONE ELSE)

Use Case A: Point your frontend to teammate API
Create/update client/.env.local:

VITE_API_URL="http://buru-degree.ts.net:5000/api"

Use Case B: Point your backend to teammate database
Create/update server/.env:

DATABASE_URL="postgresql://user:pass@buru-degree.ts.net:5432/enrollpro"

Use Case C: Use Vite proxy target instead of direct VITE_API_URL
If your frontend should keep using /api proxy, set:

VITE_PROXY_TARGET="http://buru-degree.ts.net:5000"

Notes:
- Prefer hostname-based URLs (.ts.net) for stable team connections.
- If VITE_API_URL is set, frontend requests go directly to that API base.
- If VITE_API_URL is not set, frontend falls back to /api (proxied by Vite).
- If DNS resolution fails temporarily, use teammate 100.x.x.x IP as fallback.

- Use 100.x.x.x IP only as fallback if needed.

Test-NetConnection buru-degree.ts.net -Port 5432
Invoke-RestMethod http://buru-degree.ts.net:5000/api/health

Expected API health response:
{"ok":true}


5) TROUBLESHOOTING

- API unreachable on 5000:
    Check firewall rule, backend process status, and app.listen host 0.0.0.0.

- DB unreachable on 5432:
    Check PostgreSQL service status, listen_addresses='*', and pg_hba.conf CIDR entry.

- Frontend still hitting localhost API:
    Confirm client/.env.local uses VITE_API_URL and restart Vite dev server.

- CORS errors when serving frontend from non-localhost origin:
    Update CLIENT_URL in server/.env to the actual frontend origin and restart backend.

- Hostname does not resolve:
    Confirm both devices are connected to Tailscale and MagicDNS is enabled in Tailscale settings.


6) SECURITY NOTE (IMPORTANT)

- These steps are for development on trusted Tailscale peers only.
- Do not expose PostgreSQL or API ports to the public internet.
- If temporary access is no longer needed, remove the firewall rules.
